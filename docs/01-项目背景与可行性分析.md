# 01-项目背景与可行性分析

## 1. 项目背景

随着学习、就业和生活压力增加，很多用户需要一个低压力的空间记录情绪、整理想法，并在需要时获得温和的回应。传统日记类应用偏记录，社区类应用偏公开表达，而本项目希望把“私人情绪记录”“AI 陪伴回复”和“匿名互助社区”结合起来，让用户既能保留自己的情绪轨迹，也能在匿名社区中获得轻量互动。

心屿项目以 Java Web 全栈实践为目标，基于 Spring Boot + Vue3 构建前后端分离应用。项目当前已经实现用户端、社区端、后台管理端和 AI 回复记录管理，具备较完整的实习简历展示价值。

## 2. 建设目标

系统建设目标包括：

1. 支持用户注册、登录和退出登录。
2. 支持用户记录、查看、修改、删除心情记录。
3. 支持心情记录 AI 陪伴回复。
4. 支持最近 7 天心情趋势统计。
5. 支持匿名社区发帖、回复、点赞、收藏和话题分类。
6. 支持社区帖子 AI 回复。
7. 支持管理员管理用户、帖子、回复和话题。
8. 支持管理员查看 AI 回复记录和调用状态。
9. 支持后台概览统计用户、内容和 AI 调用情况。
10. 保持前后端分离、接口规范、权限清晰和文档完整。

## 3. 项目价值

### 3.1 用户价值

对普通用户来说，系统提供：

- 私密心情记录空间
- 温和的 AI 回复
- 最近 7 天情绪变化概览
- 匿名社区表达渠道
- 点赞、收藏、回复等轻量互动

系统不定位为医疗诊断工具，也不替代专业心理咨询服务。它更适合作为情绪记录、陪伴反馈和匿名表达的辅助工具。

### 3.2 管理价值

对管理员来说，系统提供：

- 用户状态管理
- 帖子内容管理
- 回复内容管理
- 话题管理
- AI 回复记录审查
- 平台基础数据概览

这些能力让项目具备平台型应用的基本治理结构，而不只是用户端单向功能。

### 3.3 工程价值

从工程实践角度看，项目覆盖：

- Spring Boot 分层开发
- MyBatis-Plus 分页和条件查询
- JWT 登录认证
- BCrypt 密码加密
- 角色权限控制
- Vue3 页面开发
- Pinia 状态管理
- axios 请求封装
- AI 接入和失败兜底
- 后台管理和组件复用
- Markdown 项目文档

这些内容适合作为实习面试中的主项目展开讲解。

## 6. 可行性分析

### 6.1 技术可行性

项目采用成熟技术栈：

- Spring Boot 适合快速构建 REST API。
- MyBatis-Plus 适合完成分页、条件查询和常见 CRUD。
- MySQL 适合保存结构化业务数据。
- JWT 适合前后端分离登录认证。
- Vue3 + Vite 适合构建现代前端页面。
- Pinia 适合管理跨页面状态。
- DeepSeek API 可以提供 AI 回复能力。

这些技术资料丰富、生态成熟，适合个人开发和实习项目展示。

### 6.2 业务可行性

情绪记录、AI 回复、匿名社区和后台治理之间具有自然关联：

```text
记录情绪
-> 获得 AI 回复
-> 进入匿名社区表达
-> 产生帖子、回复、点赞、收藏
-> 管理员进行内容治理
-> 后台查看数据和 AI 调用记录
```

业务链路完整，功能之间不是孤立堆叠。

### 6.3 数据可行性

系统核心数据可以通过关系型数据库清晰表达：

- 用户和心情记录是一对多。
- 用户和帖子是一对多。
- 帖子和回复是一对多。
- 用户和帖子通过点赞、收藏关系表关联。
- AI 回复通过 `target_type + target_id` 关联心情记录或帖子。

这种结构易于查询、维护和扩展。

### 6.4 成本可行性

本地开发只需要：

- JDK
- Maven
- MySQL
- Node.js
- 浏览器

AI 接口可能产生少量成本，但项目已经设计失败兜底逻辑，即使 AI 服务不可用，主业务也可以继续运行。部署时可以选择低成本云服务器进行演示。

## 7. 项目风险与应对

| 风险 | 应对方式 |
| --- | --- |
| AI 服务不可用 | 返回兜底文案，记录失败日志 |
| 用户越权访问 | 后端从 token 解析 userId，不信任前端 userId |
| 普通用户访问后台 | 后端校验 `role=ADMIN` |
| 社区内容不合适 | 管理员可隐藏帖子和回复 |
| 文档落后于代码 | 阶段完成后同步 README、接口、测试和计划文档 |
| 部署环境差异 | 使用环境变量和部署说明隔离配置 |

## 10. 可行性结论

心屿项目主题明确、业务自然、技术栈成熟，已经从最初的登录注册和心情 CRUD 扩展为包含 AI、社区、后台、统计和文档的综合系统。

从用户价值看，它能够提供情绪记录、AI 陪伴和匿名互动；从工程价值看，它覆盖了实习项目中常见的认证、权限、数据库、接口、前端状态管理、后台管理和 AI 接入能力。因此该项目具备继续完善和用于简历展示的可行性。
