# 09-部署与运行说明

## 1. 文档目的

本文档用于说明“心屿 AI 情绪记录与匿名互助社区系统”的本地运行、环境配置、数据库初始化、前后端启动、构建验证和生产部署建议。

当前项目已经包含用户端、社区端、后台管理端、AI 回复记录管理和后台概览统计，因此部署说明按完整项目状态编写。

## 2. 项目运行架构

本地开发架构：

```text
浏览器
  -> Vue3 前端：http://localhost:5173
  -> Spring Boot 后端：http://localhost:8080
  -> MySQL 数据库：localhost:3306/xinyu
  -> DeepSeek API
```

运行时数据流：

```text
用户访问前端页面
-> 前端通过 request.js 调用后端接口
-> request.js 自动携带 token
-> 后端处理业务逻辑
-> 后端读写 MySQL
-> 需要 AI 回复时调用 DeepSeek API
-> 后端返回统一 Result
-> 前端渲染页面
```

## 5. 数据库初始化

### 5.1 创建数据库

```sql
CREATE DATABASE xinyu DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;
```

### 5.2 导入 SQL

导入项目脚本：

```text
sql/xinyu.sql
```

脚本应包含当前项目所需表：

- `user`
- `mood_record`
- `ai_reply`
- `topic`
- `community_post`
- `community_reply`
- `post_like`
- `favorite`

如果后续 SQL 有更新，不建议在正式库直接重复全量导入，应该根据差异执行增量 SQL。

## 6. 后端配置

后端配置文件位于：

```text
xinyu-backend/src/main/resources/application.yml
```

常见配置项：

```yaml
spring:
  datasource:
    url: ${DB_URL:jdbc:mysql://localhost:3306/xinyu}
    username: ${DB_USERNAME:root}
    password: ${DB_PASSWORD:}

deepseek:
  api-key: ${DEEPSEEK_API_KEY:}
  base-url: https://api.deepseek.com
  model: deepseek-v4-flash
```

说明：

- 数据库连接优先通过 `DB_URL`、`DB_USERNAME`、`DB_PASSWORD` 环境变量配置。
- `DEEPSEEK_API_KEY` 通过环境变量配置，不要写死到公开仓库。
- 如果没有配置 AI Key，AI 回复功能可能走失败兜底逻辑。

## 8. 前端配置


## 11. 运行状态检查

### 11.1 浏览器检查

打开 DevTools：

- `Application -> LocalStorage` 检查 token 和用户信息。
- `Network -> Fetch/XHR` 检查接口请求是否成功。
- `Console` 检查是否有前端报错。

### 11.2 数据库检查

关键表检查：

| 表 | 检查点 |
| --- | --- |
| `user` | 用户是否创建，密码是否为 BCrypt 密文 |
| `mood_record` | 心情记录和 `ai_reply` 是否保存 |
| `community_post` | 帖子和 `ai_reply` 是否保存 |
| `community_reply` | 回复是否保存 |
| `ai_reply` | AI 调用成功/失败日志是否保存 |

### 11.3 后台检查

管理员登录后检查：

- 概览统计是否有数据
- 用户列表是否正常
- 帖子筛选是否正常
- 回复筛选是否正常
- 话题新增和编辑是否正常
- AI 记录是否能查看详情

## 12. 常见问题

### 12.1 后端连接 MySQL 失败

可能原因：

- MySQL 未启动
- 数据库 `xinyu` 不存在
- 数据库用户名或密码错误
- JDBC URL 参数缺失

处理方式：

- 启动 MySQL
- 创建并导入 `xinyu` 数据库
- 检查 `application.properties`
- 确认 URL 中包含 `allowPublicKeyRetrieval=true`

### 12.2 前端请求失败

可能原因：

- 后端未启动
- `VITE_API_BASE_URL` 配置错误
- 浏览器缓存旧环境变量
- token 失效

处理方式：

- 确认后端端口为 `8080`
- 修改 `.env.development` 后重启前端
- 清理 LocalStorage 后重新登录
- 在 Network 中查看真实请求地址

### 12.3 AI 回复为空或为兜底文案

可能原因：

- `DEEPSEEK_API_KEY` 未配置
- AI 服务请求失败
- 网络不可用
- 第三方接口返回异常

处理方式：

- 检查环境变量
- 查看后端日志
- 查看 `ai_reply.error_message`
- 确认主业务是否已经保存成功

### 12.4 后台页面无权限

可能原因：

- 当前账号不是管理员
- token 对应用户角色不是 `ADMIN`
- localStorage 中保存了旧用户信息

处理方式：

- 使用管理员账号登录
- 检查 `user.role`
- 清理 LocalStorage 后重新登录

## 13. 生产部署建议

### 13.1 部署结构

```text
用户浏览器
  -> Nginx
     -> Vue dist 静态资源
     -> /api 反向代理到 Spring Boot
  -> MySQL
```

后续如果需要性能优化，可以加入 Redis：

```text
Spring Boot
  -> MySQL
  -> Redis
  -> DeepSeek API
```

### 13.2 前端部署

构建：

```bash
cd xinyu-vue
npm run build
```

构建产物：

```text
xinyu-vue/dist
```

将 `dist` 部署到 Nginx 静态资源目录。

### 13.3 后端部署

打包：

```bash
cd xinyu-backend
mvn clean package
```

运行：

```bash
java -jar target/xinyu-backend-0.0.1-SNAPSHOT.jar
```

生产环境建议通过环境变量配置：

- 数据库地址
- 数据库用户名
- 数据库密码
- JWT 密钥
- DeepSeek API Key

### 13.4 Nginx 示例

```nginx
server {
    listen 80;
    server_name your-domain.com;

    root /usr/share/nginx/html/xinyu;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://127.0.0.1:8080/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
```

前端生产环境 `VITE_API_BASE_URL` 配置为：

```text
/api
```

### 13.5 数据库部署建议

生产环境建议：

- 不使用 root 账号连接业务系统
- 使用强密码
- 限制远程访问
- 定期备份数据库
- 不把数据库密码提交到 Git

## 14. 部署与运行结论

心屿项目本地运行依赖 JDK、Maven、MySQL、Node.js 和浏览器。开发环境下，前端运行在 `5173`，后端运行在 `8080`，数据库运行在 `3306`。

当前项目已经具备完整业务闭环，部署时需要重点保证数据库脚本、环境变量、前端接口地址、后端服务端口和 Nginx 代理配置一致。后续如果继续增强，可以加入 Redis 缓存、日志文件、进程守护和自动化部署。
