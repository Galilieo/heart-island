# 文档导航

`docs` 按资料用途拆分，避免把当前项目事实、历史计划和个人求职话术混在一起。

## 目录结构

```text
docs/
├─ project/    当前项目交付文档，可公开
├─ history/    历史设计规格与实施计划，可公开但不代表当前实现
└─ personal/   个人求职、面试、学习和私人运维资料，仅本地使用
```

## 项目文档

`project/` 包含需求、架构、数据库、接口、详细设计、前端设计、测试、部署、总结和阶段计划。

项目技术事实的判断顺序：

```text
当前源码、SQL、配置
→ 当前构建和测试结果
→ Git 历史
→ project/ 当前状态文档
→ history/ 历史规格与计划
```

文档与源码冲突时，以当前源码和实际验证结果为准。

## 历史资料

`history/` 保存曾用于开发的设计规格和实施计划。它们用于理解项目演进，不保证其中每项计划都已实现，也可能包含移动前的历史命令或路径。

## 个人资料

`personal/` 已由 `.gitignore` 整体忽略，不会提交到公开仓库。

```text
personal/
├─ context/    跨对话个人、项目与求职总上下文
├─ interview/  面试题库、答案册和历史面试稿
├─ career/     简历、求职方案和岗位路线
├─ learning/   学习计划与知识掌握统计
└─ private/    私人部署和运维笔记
```

个人资料中的项目说法可能是历史快照或面试话术，不能覆盖当前源码事实。`private/` 中的内容不得公开或复制到外部回复。
