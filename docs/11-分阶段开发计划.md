# 分阶段开发计划

## 1. 文档目的

本文档用于记录“心屿 AI 情绪记录与匿名互助社区系统”的阶段开发路线、当前完成情况和后续优先级。

项目最初从注册登录和心情记录开始，逐步扩展到 AI 回复、匿名社区、后台管理、AI 调用记录和平台统计。当前版本已经完成主要业务闭环，后续重点应从“继续堆功能”转向“完善展示质量、测试验证和部署上线”。

## 2. 当前总体状态

截至当前版本，项目已经完成以下主线：

- 用户注册、登录、JWT 鉴权、角色权限
- 心情记录新增、查询、详情、修改、删除、筛选、分页
- 心情记录 AI 回复生成、展示和日志保存
- 最近 7 天情绪趋势
- 匿名社区话题、帖子、回复、点赞、收藏
- 社区帖子 AI 回复
- 后台用户管理、帖子管理、回复管理、话题管理
- 后台 AI 回复记录管理
- 后台概览统计，包括用户、内容和 AI 调用统计
- Vue 前端基础组件和后台公共组件拆分

因此，项目已经具备较完整的简历展示价值。

## 3. 已完成阶段

### 阶段 0：项目基础搭建

完成内容：

- Spring Boot 后端项目搭建
- Vue3 前端项目搭建
- MySQL 数据库脚本
- axios 请求封装
- Pinia 状态管理
- Vue Router 路由管理
- 基础 UI 组件

完成标准：

- 前后端可以独立启动
- 前端可以正常请求后端接口
- 数据库连接正常

当前状态：已完成。

### 阶段 1：用户与心情记录基础闭环

完成内容：

- 用户注册
- 用户登录
- JWT token 生成和校验
- BCrypt 密码加密
- 登录后保存用户基础信息
- 心情记录 CRUD
- 心情记录分页、筛选和详情

完成标准：

- 用户登录后可以完整管理自己的心情记录
- 未登录用户无法访问受保护接口
- 用户不能操作他人的私有记录

当前状态：已完成。

### 阶段 2：AI 情绪陪伴能力

完成内容：

- DeepSeek API 接入
- 新增心情记录后生成 AI 回复
- 修改心情记录后重新生成 AI 回复
- AI 回复写入 `mood_record.ai_reply`
- AI 调用记录写入 `ai_reply` 表
- AI 失败时返回兜底文案
- 最近 7 天情绪趋势统计

完成标准：

- AI 成功时用户能看到温和的陪伴回复
- AI 失败时主业务不失败
- 后台可以追踪 AI 调用记录

当前状态：已完成。

### 阶段 3：匿名互助社区

完成内容：

- 话题列表
- 帖子列表和详情
- 发帖、编辑帖子、删除帖子
- 回复帖子、编辑回复、删除回复
- 点赞和取消点赞
- 收藏和取消收藏
- 收藏列表
- 最新和热门排序
- 帖子 AI 回复生成与展示

完成标准：

- 用户可以围绕话题匿名发布内容
- 用户可以与帖子互动
- 帖子详情能看到社区内容、回复和 AI 心屿回复

当前状态：已完成。

### 阶段 4：后台管理与内容治理

完成内容：

- 后台入口和后台路由
- 管理员角色权限控制
- 后台概览页
- 用户分页、搜索、状态筛选、启用/禁用
- 帖子分页、内容搜索、帖子 id、用户 id、状态筛选、隐藏/恢复
- 回复分页、内容搜索、回复 id、帖子 id、用户 id、状态筛选、隐藏/恢复
- 话题新增、编辑、启用/禁用、排序
- AI 回复记录分页、筛选和详情查看

完成标准：

- 普通用户不能访问后台管理接口
- 管理员可以完成基础内容治理
- 后台页面具备列表、筛选、分页和状态操作

当前状态：已完成。

### 阶段 5：后台统计与前端组件整理

完成内容：

- 用户总数、禁用用户数统计
- 帖子总数、隐藏帖子数统计
- 回复总数、隐藏回复数统计
- AI 调用总数、成功次数、失败次数统计
- AI 失败率展示
- 后台页头组件 `AdminPageHeader`
- 后台筛选按钮组件 `AdminFilterActions`
- 后台标签导航组件 `AdminTabs`

完成标准：

- 管理员进入后台可以快速了解平台状态
- 后台多个页面的筛选和页头风格统一
- 页面组件拆分不会破坏现有功能

当前状态：已完成。

## 4. 当前最应该做的阶段

### 阶段 6：文档、测试和部署完善

这是当前最值得做的阶段。

原因：

- 项目核心功能已经足够完整。
- 简历项目不只是看功能数量，也看工程表达、文档完整度和可运行性。
- 面试时经常会问“你怎么测试”“怎么部署”“遇到 AI 失败怎么办”“权限怎么做”。
- 文档和测试可以把已经做好的功能变成能讲清楚的项目亮点。

建议任务：

1. 同步 README、接口文档、测试文档和阶段计划。
2. 用管理员和普通用户各走一遍完整流程。
3. 补充关键接口测试截图或记录。
4. 准备部署说明和服务器部署速查。
5. 整理简历描述和面试讲解口径。

完成标准：

- README 与实际功能一致。
- 接口文档中的路径与代码一致。
- 测试文档覆盖用户端、社区、后台和 AI。
- 项目可以通过 `mvn test` 和 `npm run build`。
- 能够用 2 到 3 分钟讲清楚项目亮点。

建议优先级：中。

说明：如果准备把项目部署到服务器展示，这部分优先级会上升。

## 7. 面试讲解建议

可以按这个顺序介绍项目：

1. 项目解决什么问题：情绪记录、AI 陪伴、匿名互助。
2. 用户端有什么：心情记录、AI 回复、社区互动。
3. 管理端有什么：用户、帖子、回复、话题、AI 记录和概览统计。
4. 后端怎么设计：Controller、Service、Mapper、统一返回、JWT、角色权限。
5. AI 怎么接入：同步调用、失败兜底、日志留痕、targetType 支持多业务。
6. 前端怎么组织：Vue3、Pinia、路由、api 分层、后台公共组件。
7. 后续怎么优化：图表、举报、Redis、部署和测试。

## 8. 阶段结论

心屿项目已经完成从用户端到管理端的主要闭环，不再处于“还缺核心模块”的阶段。下一步不应该盲目新增页面，而应该优先提升项目的可展示性、可验证性和可讲述性。

对于简历和实习来说，当前最有价值的方向是：

- 文档保持同步
- 核心流程稳定
- 后台和 AI 亮点讲清楚
- 部署和测试补齐
