# 测试说明书

## 1. 文档目的

本文档用于说明“心屿 AI 情绪记录与匿名互助社区系统”的测试范围、测试方法、核心用例和验收标准。

当前项目已经完成用户端、社区端、后台管理端和 AI 回复记录管理，因此测试重点不再只是注册登录和心情 CRUD，而是要验证完整业务闭环是否稳定。

## 2. 测试目标

测试目标包括：

1. 验证用户注册、登录、退出和 token 携带是否正常。
2. 验证心情记录 CRUD、筛选、分页和详情是否正常。
3. 验证心情记录 AI 回复生成、展示和日志保存是否正常。
4. 验证匿名社区发帖、回复、点赞、收藏是否正常。
5. 验证社区帖子 AI 回复是否正常生成和展示。
6. 验证管理员后台用户、帖子、回复、话题管理是否正常。
7. 验证后台 AI 回复记录筛选、分页和详情是否正常。
8. 验证后台概览统计是否正确展示。
9. 验证普通用户不能访问管理员接口。
10. 验证前端构建和后端测试可以通过。

## 4. 自动化验证命令

后端测试：

```bash
cd xinyu-backend
mvn test
```

前端构建：

```bash
cd xinyu-vue
npm run build
```

说明：

- `mvn test` 用于验证后端基础测试和编译。
- `npm run build` 用于验证前端语法、依赖引用和生产构建。
- 文档更新不需要每次运行构建，但功能提交前建议执行。

## 12. 验收标准

项目当前阶段验收标准：

1. 前后端可以正常启动。
2. `mvn test` 可以通过。
3. `npm run build` 可以通过。
4. 普通用户可以完成注册、登录、心情记录和社区互动。
5. 心情记录和帖子都可以生成 AI 回复。
6. AI 失败不阻断心情或帖子保存。
7. AI 调用记录可以在后台查看。
8. 管理员可以管理用户、帖子、回复和话题。
9. 后台概览可以展示用户、内容和 AI 统计。
10. 普通用户不能访问后台接口。
11. README、接口文档和阶段计划与实际功能一致。

## 13. 测试结论

当前测试重点应围绕“用户端完整体验 + 后台治理 + AI 留痕”展开。只要这些流程稳定，项目就已经具备较好的简历展示价值。后续如果继续增强，可补充自动化接口测试、E2E 测试和线上部署验证。
