# 心屿 AI 情绪记录与匿名互助社区系统

# 09-部署与运行说明

## 1. 文档目的

本文档用于说明心屿 AI 情绪记录与匿名互助社区系统的本地运行、环境配置、数据库初始化、前端启动、后端启动、接口联调和后续服务器部署方案。

部署与运行说明既服务于开发者本地调试，也服务于后续项目开源、演示和上线部署。

## 2. 项目运行架构

系统采用前后端分离架构：

```text
浏览器
  -> Vue 前端服务：http://localhost:5173
  -> Spring Boot 后端服务：http://localhost:8080
  -> MySQL 数据库：localhost:3306
```

运行时数据流：

```text
用户访问前端页面
-> 前端通过 request.js 调用后端接口
-> 后端处理业务逻辑
-> 后端访问 MySQL 数据库
-> 后端返回统一 Result
-> 前端渲染页面
```

## 5. 数据库初始化

### 5.1 创建数据库

登录 MySQL 后执行：

```sql
CREATE DATABASE xinyu DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;
```

### 5.2 使用数据库

```sql
USE xinyu;
```

### 5.3 创建用户表

```sql
CREATE TABLE user (
  id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '用户id',
  username VARCHAR(50) NOT NULL COMMENT '用户名',
  password VARCHAR(100) NOT NULL COMMENT '加密后的密码',
  nickname VARCHAR(50) NOT NULL COMMENT '昵称',
  avatar VARCHAR(255) DEFAULT NULL COMMENT '头像地址',
  role VARCHAR(20) NOT NULL DEFAULT 'USER' COMMENT '角色：USER/ADMIN',
  status TINYINT NOT NULL DEFAULT 1 COMMENT '状态：1正常，0禁用',
  create_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  UNIQUE KEY uk_user_username (username)
) COMMENT='用户表';
```

### 5.4 创建心情记录表

```sql
CREATE TABLE mood_record (
  id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '心情记录id',
  user_id BIGINT NOT NULL COMMENT '用户id',
  mood_type VARCHAR(30) NOT NULL COMMENT '心情类型',
  mood_score TINYINT DEFAULT NULL COMMENT '心情分数',
  content TEXT NOT NULL COMMENT '心情内容',
  ai_reply TEXT DEFAULT NULL COMMENT 'AI回复内容',
  is_public TINYINT NOT NULL DEFAULT 0 COMMENT '是否公开：0私密，1公开',
  create_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  KEY idx_mood_user_time (user_id, create_time),
  KEY idx_mood_user_type (user_id, mood_type)
) COMMENT='心情记录表';
```

## 6. 后端配置

### 6.1 application.properties 示例

后端配置文件位于：

```text
src/main/resources/application.properties
```

示例配置：

```properties
spring.application.name=xinyu-backend

spring.datasource.driver-class-name=com.mysql.cj.jdbc.Driver
spring.datasource.url=jdbc:mysql://localhost:3306/xinyu?serverTimezone=Asia/Shanghai&useUnicode=true&characterEncoding=utf8&useSSL=false&allowPublicKeyRetrieval=true
spring.datasource.username=root
spring.datasource.password=123456
```

### 6.2 配置说明

| 配置项 | 说明 |
| --- | --- |
| spring.datasource.url | MySQL 连接地址 |
| spring.datasource.username | 数据库用户名 |
| spring.datasource.password | 数据库密码 |
| allowPublicKeyRetrieval=true | MySQL 8 认证常用参数 |
| serverTimezone=Asia/Shanghai | 设置数据库时区 |

### 6.3 注意事项

1. 数据库名需要与配置中的 xinyu 一致。
2. 数据库用户名和密码需要与本机 MySQL 一致。
3. MySQL 8 如果出现 CachingSha2PasswordPlugin 相关错误，通常需要检查 allowPublicKeyRetrieval=true。
4. 修改配置后需要重启后端服务。

## 8. 前端配置

### 8.1 request.js 配置

前端统一请求文件：

```text
src/utils/request.js
```

示例：

```js
import axios from 'axios'

const request = axios.create({
  baseURL: 'http://localhost:8080',
  timeout: 5000
})

request.interceptors.request.use(config => {
  const token = localStorage.getItem('token')

  if (token) {
    config.headers.token = token
  }

  return config
})

export default request
```

### 8.2 配置说明

1. baseURL 指向后端服务地址。
2. timeout 设置请求超时时间。
3. 请求拦截器自动从 localStorage 读取 token。
4. token 存在时加入请求头。

## 10. 前后端联调流程

### 10.1 注册流程

1. 打开前端注册页。
2. 输入 username、password、nickname。
3. 点击注册。
4. 前端调用 /user/register。
5. 后端保存用户。
6. 注册成功后跳转登录页。

### 10.2 登录流程

1. 打开登录页。
2. 输入 username 和 password。
3. 点击登录。
4. 前端调用 /user/login。
5. 后端返回 user 和 token。
6. 前端保存 loginUser 和 token。
7. 跳转首页。

### 10.3 心情记录流程

1. 用户登录后进入首页。
2. 输入心情类型和内容。
3. 点击新增心情记录。
4. 前端调用新增接口。
5. request.js 自动携带 token。
6. 后端从 token 解析 userId。
7. 后端保存心情记录。
8. 前端刷新列表。

### 10.4 退出登录流程

1. 用户点击退出登录。
2. 前端删除 localStorage 中的 loginUser 和 token。
3. 页面跳转登录页。

```text
Application -> LocalStorage -> http://localhost:5173
```

```text
loginUser

```

2. token 是一长串 JWT 字符串。

### 11.2 Network 验证

打开浏览器 DevTools：

```text
Network -> Fetch/XHR
```

点击刷新心情记录，查看 /mood/list 请求：

```text
Request Headers 中应包含 token
```

### 11.3 数据库验证

1. 注册用户后，user 表应新增一条记录。
2. password 字段应为 BCrypt 密文。
3. 新增心情记录后，mood_record 表应新增一条记录。
4. mood_record.user_id 应等于当前登录用户 id。

## 12. 常见问题

### 12.1 后端连接 MySQL 失败

可能原因：

1. MySQL 服务未启动。
2. 数据库 xinyu 不存在。
3. application.properties 中用户名或密码错误。
4. MySQL 8 认证需要 allowPublicKeyRetrieval=true。

处理方式：

1. 启动 MySQL。
2. 创建 xinyu 数据库。
3. 检查 username 和 password。
4. 检查 JDBC URL 是否包含 allowPublicKeyRetrieval=true。

### 12.2 Apifox 返回 400 Bad Request

可能原因：

1. JSON 格式错误。
2. 字段名没有加双引号。
3. 字符串值没有加双引号。
4. JSON 少逗号或多逗号。

正确示例：

```json
{
  "username": "test",
  "password": "123456"
}
```

### 12.3 Apifox 返回 500 Internal Server Error

可能原因：

1. 后端代码抛异常。
2. 数据库连接失败。
3. 表不存在。
4. 字段名和数据库不一致。
5. token 解析失败。

处理方式：

1. 查看 IDEA 控制台异常。
2. 找到 Caused by。
3. 根据异常定位数据库、代码或参数问题。

### 12.4 前端查不到心情记录

可能原因：

1. 当前用户没有记录。
2. token 对应的 userId 与旧数据 user_id 不一致。
3. 请求没有携带 token。
4. 后端没有启动或接口报错。

处理方式：

1. 使用当前登录用户新增一条记录。
2. 在 Network 检查请求头是否有 token。
3. 查看后端控制台。
4. 查看 mood_record 表 user_id。

### 12.5 页面显示乱码

可能原因：

1. 文件编码不是 UTF-8。
2. 编辑器编码设置不一致。

处理方式：

1. 确认文件使用 UTF-8。
2. IDEA 或 VS Code 中修改文件编码。
3. 避免复制乱码文本。

## 13. 生产部署方案

### 13.1 部署目标架构

后续可采用以下部署结构：

```text
用户浏览器
  -> Nginx
      -> 前端静态资源
      -> 后端 Spring Boot 服务
  -> MySQL
  -> Redis
```

### 13.2 前端部署

构建前端：

```bash
npm run build
```

构建结果位于：

```text
dist
```

将 dist 文件部署到 Nginx 静态资源目录。

### 13.3 后端部署

后端打包：

```bash
mvn clean package
```

运行 jar：

```bash
java -jar xinyu-backend-0.0.1-SNAPSHOT.jar
```

### 13.4 Nginx 反向代理示例

```nginx
server {
    listen 80;
    server_name example.com;

    location / {
        root /usr/share/nginx/html/xinyu;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://127.0.0.1:8080/;
    }
}
```

前端生产环境 baseURL 可配置为：

```text
/api
```

### 13.5 数据库部署

生产环境建议：

1. 创建独立数据库用户。
2. 不使用 root 账号连接业务系统。
3. 设置强密码。
4. 定期备份数据库。
5. 限制数据库远程访问权限。

### 13.6 配置安全

生产环境不建议将敏感配置直接写死在代码中。

建议通过：

1. 环境变量。
2. application-prod.properties。
3. 服务器配置文件。
4. CI/CD Secret。

管理数据库密码、JWT 密钥和 AI API Key。

## 15. 部署与运行结论

心屿系统本地运行主要依赖 JDK、Maven、MySQL、Node.js 和浏览器。开发阶段前端运行在 5173 端口，后端运行在 8080 端口，数据库运行在 3306 端口。通过 CORS 配置和 request.js 请求封装，前后端可以完成正常联调。

后续部署时，可以使用 Nginx 部署前端静态资源，并通过反向代理访问后端 Spring Boot 服务。数据库和 Redis 应独立配置，并注意账号权限、密码安全、数据备份和生产环境配置隔离。
