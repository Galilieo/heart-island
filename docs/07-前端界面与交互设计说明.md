# 心屿 AI 情绪记录与匿名互助社区系统

# 07-前端界面与交互设计说明

## 1. 文档目的

本文档用于说明心屿 AI 情绪记录与匿名互助社区系统的前端界面设计和交互设计方案，包括整体视觉风格、页面结构、用户操作流程、组件规划、状态反馈、异常提示、响应式设计和后续扩展方向。

前端设计以产品体验为出发点，同时结合软件工程中的组件化、可维护性和前后端联调规范，保证页面既能清晰表达业务，也便于后续功能扩展。

## 2. 前端设计目标

前端设计需要满足以下目标：

1. 提供清晰、低压力的情绪记录体验。
2. 让用户能够快速完成注册、登录、记录心情、查看记录和社区互动。
3. 通过温和的视觉风格强化系统的陪伴感。
4. 通过统一组件和请求封装降低维护成本。
5. 支持用户端和管理端两类页面。
6. 支持已完成的 AI 回复、匿名社区、后台管理和后续数据统计扩展。

## 3. 整体视觉风格

### 3.1 产品气质

心屿的产品气质应以“温和、安静、可信、低压力”为主。界面不应给用户强烈压迫感，也不应使用过于商业化或刺激性的视觉表达。

建议关键词：

1. 温和。
2. 安静。
3. 陪伴感。
4. 私密感。
5. 轻量。
6. 清晰。

### 3.2 色彩风格

用户端采用“奶油暖白 + 薄荷青主色 + 杏粉点缀”的柔暖手绘方向，并将这套配色固化为全局设计 token（`src/styles/tokens.css`），所有页面统一引用。

配色基准：

1. 背景色：奶油暖白，叠加细微纸纹理与暖色光晕，降低冷感与压迫感。
2. 主色：薄荷青，用于品牌标识、主按钮、选中状态和主要操作。
3. 点缀色：杏粉色，用于强调、AI 回复区域和卡片装饰，避免大面积使用。
4. 心情色：开心、平静、难过、生气、疲惫各有一套柔和色，用于心情标签与趋势图。
5. 成功色柔和绿、警告色暖黄、危险色低饱和暖红。
6. 文本色为暖黑而非纯黑，边框使用米色而非冷灰。
7. 阴影采用暖色调，圆角偏大（12 至 24px），整体偏圆润手绘感。

### 3.3 页面氛围

用户端页面避免复杂布局，优先使用卡片、留白、圆角和柔和阴影，并以少量手绘 SVG 涂鸦（云、叶、星、波浪等）点缀，让用户专注于表达情绪。

管理端页面应更偏工具化，使用清晰的表格、筛选栏、状态标签和操作按钮，重点保证管理效率。

当前所有用户端页面统一采用同一套设计系统：奶油暖白背景、薄荷青主色、杏粉点缀、大圆角卡片和暖色阴影。交互反馈使用全局 Toast 提示、卡片式确认弹窗和加载骨架屏，避免浏览器原生 alert 带来的强打断感。

## 4. 前端技术结构

### 4.1 技术选型

| 技术 | 用途 |
| --- | --- |
| Vue3 | 构建页面和组件，使用 `<script setup>` 组合式 API |
| Vue Router | 页面路由跳转与登录守卫 |
| Pinia | 全局状态管理（用户、心情、社区） |
| axios | 请求后端接口 |
| request.js | axios 二次封装，统一携带 token、统一响应处理 |
| localStorage | 保存 loginUser 和 token |
| Vite | 开发与构建工具 |
| ECharts | 后续情绪统计和平台统计图表 |

前端不使用第三方 UI 组件库，自建设计 token 与基础组件库。

实际前端目录结构：

```text

│  ├─ mood        心情业务组件
│  └─ community   社区业务组件
├─ router
│  └─ index.js
├─ utils
│  ├─ request.js
│  └─ mood.js     心情常量与工具函数
├─ views
│  ├─ Login.vue
│  ├─ Register.vue
│  ├─ Home.vue
│  ├─ PostList.vue
│  ├─ PostDetail.vue

│  └─ Profile.vue

└─ main.js
```

管理端页面当前放在 `src/views` 下，并通过公共后台组件保持风格统一：


## 5. 路由设计

### 5.1 用户端路由

| 路由 | 页面 | 是否需要登录 | 说明 |
| --- | --- | --- | --- |
| / | Login.vue | 否 | 登录页 |
| /register | Register.vue | 否 | 注册页 |
| /home | Home.vue | 是 | 首页，记录与查看心情、近 7 天趋势 |
| /posts | PostList.vue | 是 | 社区列表页，发帖表单内嵌于本页 |
| /community/post/:id | PostDetail.vue | 是 | 帖子详情页，含回复 |
| /favorites | Favorites.vue | 是 | 个人收藏页 |
| /profile | Profile.vue | 是 | 个人资料页，修改昵称与密码 |

说明：心情详情与编辑通过首页弹窗实现，未单独成页；情绪统计页属于后续数据统计阶段。

### 5.2 管理端路由

| 路由 | 页面 | 是否需要管理员 |
| --- | --- | --- |
| /admin | AdminDashboard.vue | 是 |
| /admin/users | AdminUsers.vue | 是 |
| /admin/posts | AdminPosts.vue | 是 |
| /admin/replies | AdminReplies.vue | 是 |
| /admin/topics | AdminTopics.vue | 是 |
| /admin/ai-replies | AdminAiReplies.vue | 是 |

### 5.3 路由守卫设计

当前已通过 router.beforeEach 处理登录状态和后台入口权限：

```text
进入需要登录的页面
-> 检查 localStorage 中是否存在 token
-> 不存在则跳转登录页
-> 存在则允许进入
```

管理员页面还需要校验 role：

```text
进入管理员页面
-> 检查 token
-> 检查 loginUser.role 是否为 ADMIN
-> 否则跳转无权限页或登录页
```

## 6. 登录页设计

### 6.1 页面目标

登录页用于用户输入用户名和密码，完成登录并进入系统首页。

### 6.2 页面元素

1. 项目名称。
2. 简短说明文案。
3. 用户名输入框。
4. 密码输入框。
5. 登录按钮。
6. 注册入口。
7. 错误提示。
8. 登录中 loading 状态。

### 6.3 交互流程

```text
用户输入用户名和密码
-> 点击登录
-> 前端校验是否为空
-> 调用 /user/login
-> 登录成功，保存 loginUser 和 token
-> 跳转 /home
-> 登录失败，显示错误提示
```

### 6.4 状态设计

| 状态 | 表现 |
| --- | --- |
| 初始状态 | 输入框为空，按钮可点击 |
| 输入中 | 用户输入账号密码 |
| 提交中 | 登录按钮显示“登录中...”并禁用 |
| 成功 | 跳转首页 |
| 失败 | 展示错误提示 |

当前实现中，登录页与首页保持同一套用户端视觉风格。用户名为空、密码为空、登录失败和服务异常均使用页面内 message 提示。

## 7. 注册页设计

### 7.1 页面目标

注册页用于新用户创建账号。

### 7.2 页面元素

1. 用户名输入框。
2. 密码输入框。
3. 昵称输入框。
4. 注册按钮。
5. 返回登录入口。
6. 错误提示。
7. 注册中 loading 状态。

### 7.3 交互流程

```text
用户填写注册信息
-> 点击注册
-> 前端校验字段是否为空
-> 调用 /user/register
-> 注册成功，提示成功并跳转登录页
-> 注册失败，展示失败原因
```

当前实现中，注册页与登录页保持一致的视觉和反馈方式。用户名、密码、昵称为空，以及注册失败和服务异常，均使用页面内 message 提示；注册按钮在请求中显示“注册中”，避免重复提交。

### 7.4 校验规则

1. 用户名不能为空。
2. 密码不能为空。
3. 昵称不能为空。
4. 密码长度可限制为 6 到 20 位。

## 8. 首页设计

### 8.1 页面目标

首页用于承载当前阶段的核心闭环：欢迎用户、新增心情记录、查看心情记录列表、删除记录和退出登录。

### 8.2 页面结构

首页建议分为三块：

1. 顶部用户区域。
2. 新增心情记录区域。
3. 我的心情记录列表区域。

### 8.3 顶部用户区域

展示内容：

1. 项目名称。
2. 欢迎语。
3. 当前用户昵称。
4. 退出登录按钮。

交互：

1. 点击退出登录。
2. 清除 localStorage 中的 loginUser 和 token。
3. 跳转登录页。

### 8.4 新增心情记录区域

表单字段：

1. 心情类型。
2. 心情内容。
3. 后续可增加心情分数。
4. 后续可增加是否公开。

交互流程：

```text
用户输入心情类型和内容
-> 点击新增
-> 校验是否为空
-> 调用新增接口
-> 新增成功后清空表单
-> 自动刷新记录列表
```

### 8.5 心情记录列表区域

展示内容：

1. 心情类型标签。
2. 心情内容。
3. 记录时间。
4. 删除按钮。
5. 后续可增加 AI 回复摘要。

交互：

1. 点击刷新，重新请求列表。
2. 点击删除，弹出确认提示。
3. 删除成功后刷新列表。
4. 点击记录卡片，后续可进入详情页。

### 8.6 空状态设计

当用户暂无心情记录时，页面不应只显示空白，而应展示友好提示。

示例：

```text
暂无心情记录
写下第一条记录，给今天的情绪一个出口。
```

### 8.7 心情标签设计

不同心情类型可使用不同标签样式：

| 心情类型 | 样式方向 |
| --- | --- |
| 开心 | 柔和蓝紫 |
| 平静 | 柔和绿色 |
| 焦虑 | 柔和橙色 |
| 难过 | 灰蓝色 |
| 其他 | 默认浅紫色 |

## 16. 组件设计

前端组件分为三层：ui 基础组件、layout 布局组件、业务组件。

### 16.1 ui 基础组件

无业务含义、全站复用：

1. BaseButton：按钮，支持多种变体、尺寸与加载态。
2. BaseCard：卡片容器，支持留白、色调与悬停。
3. BaseInput / BaseTextarea / BaseSelect：表单输入控件。
4. BaseTag：标签，内置心情配色映射。
5. BaseDialog：弹窗。
6. BasePagination：分页器。
7. EmptyState：空状态，配涂鸦插画与提示文案。
8. SkeletonCard：加载骨架屏。
9. Toast / ConfirmDialog：全局提示与确认弹窗的渲染组件。
10. Doodle：手绘风 SVG 装饰图标。

### 16.2 layout 布局组件

1. AppShell：登录后页面外壳，含背景、导航、页脚。
2. AppHeader：顶部导航，含 Logo、心情/社区/收藏导航、用户菜单。
3. AppFooter：页脚。
4. AuthLayout：登录、注册页的两栏布局。

### 16.3 业务组件

心情域：MoodCard（心情卡片）、MoodForm（记录表单）、MoodFilterBar（筛选栏）、MoodTrendPanel（趋势面板）、MoodDetailDialog（详情弹窗）。

社区域：PostCard（帖子卡片）、PostPublishForm（发帖表单）、PostReplyForm（回复表单）、PostReplyList（回复列表）、PostEditDialog（帖子编辑弹窗）、TopicFilter（话题筛选）。

## 17. 交互状态设计

### 17.1 加载状态

接口请求过程中应展示加载状态，避免用户重复点击。

示例：

1. 登录按钮显示“登录中...”。
2. 注册按钮显示“注册中”。
3. 新增按钮显示“提交中...”。
4. 列表区域显示 loading。

### 17.2 成功状态

操作成功后应给出明确反馈。

示例：

1. 注册成功。
2. 登录成功。
3. 新增成功。
4. 删除成功。
5. 发布成功。

### 17.3 失败状态

操作失败时应展示后端返回的 message。

示例：

1. 用户名已存在。
2. 账号或密码错误。
3. 请先登录。
4. 权限不足。
5. 服务异常，请稍后再试。

### 17.4 确认状态

危险操作前应二次确认。

适用场景：

1. 删除心情记录。
2. 删除帖子。
3. 删除评论。
4. 禁用用户。

## 18. 响应式设计

### 18.1 桌面端

桌面端可以使用双栏布局：

1. 左侧为新增心情区域。
2. 右侧为心情记录列表。

社区页面可以使用列表 + 侧边话题栏。

管理端使用表格布局。

### 18.2 移动端

移动端应使用单栏布局：

1. 表单区域和列表区域上下排列。
2. 按钮宽度适当增大。
3. 卡片间距适当缩小。
4. 表格类管理页面可横向滚动或简化展示字段。

## 19. 前后端联调设计

### 19.1 登录联调

```text
前端调用 /user/login
-> 后端返回 loginUser 和 token
-> 前端保存 localStorage
-> 跳转 /home
```

### 19.2 token 携带

```text
页面调用 request
-> request.js 从 localStorage 获取 token
-> 自动加入请求头
-> 后端解析 token
```

### 19.3 心情记录联调

```text
新增心情
-> 调用新增接口
-> 新增成功
-> 清空表单
-> 刷新列表
```

### 19.4 退出登录联调

```text
点击退出登录
-> 清除 loginUser 和 token
-> 跳转登录页
```

## 20. 前端设计结论

心屿系统前端设计以“温和、安静、低压力、陪伴感”为用户端核心风格，以“清晰、效率、工具化”为管理端核心风格。用户端重点保证记录心情、查看历史、获得 AI 回复和社区互动的体验流畅；管理端重点保证内容审核、用户管理和数据统计的操作效率。

通过 Vue3 页面组件化、Vue Router 路由管理、request.js 请求封装、localStorage 登录状态保存、空状态、加载状态和后台公共组件设计，前端已经支撑用户端、匿名社区、AI 回复展示和后台管理的主要闭环。后续可以继续在图表统计、举报处理和更细粒度权限体验上增强。
