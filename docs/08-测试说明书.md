# 心屿 AI 情绪记录与匿名互助社区系统

# 08-测试说明书

## 1. 文档目的

本文档用于说明心屿 AI 情绪记录与匿名互助社区系统的测试方案，包括测试目标、测试范围、测试环境、测试方法、功能测试用例、接口测试用例、权限测试、异常测试、前后端联调测试和验收标准。

测试设计以产品功能是否可用为核心，同时结合软件工程中的边界条件、异常路径、权限控制和数据安全要求，保证系统能够稳定支撑用户注册登录、心情记录、AI 回复、匿名社区、后台管理和数据统计等功能。

## 2. 测试目标

系统测试需要达到以下目标：

1. 验证用户注册、登录、退出登录流程是否正常。
2. 验证密码加密、token 生成、token 携带和 token 解析是否正常。
3. 验证心情记录新增、查询、修改、删除功能是否正常。
4. 验证用户只能访问自己的私有数据。
5. 验证 AI 回复生成和保存流程是否正常。
6. 验证匿名社区帖子、评论、点赞、收藏功能是否正常。
7. 验证管理员后台管理和内容审核功能是否正常。
8. 验证统计接口返回数据是否准确。
9. 验证前端页面和后端接口联调是否顺畅。
10. 验证异常场景下系统能返回明确提示。

## 5. 测试方法

### 5.1 功能测试

通过页面操作验证系统功能是否符合需求。

示例：

1. 在注册页注册账号。
2. 在登录页登录账号。
3. 在首页新增心情记录。
4. 查看记录列表。
5. 删除记录。
6. 退出登录。

### 5.2 接口测试

通过 Apifox 调用后端接口，验证请求参数、响应结果和异常提示是否正确。

### 5.3 联调测试

通过前端页面触发请求，在浏览器 Network 中查看接口请求和响应，验证前后端联调是否正常。

### 5.4 权限测试

通过不携带 token、携带错误 token、普通用户访问管理员接口等方式，验证权限控制是否有效。

### 5.5 数据库验证

通过 MySQL 客户端查看数据库数据，验证注册、心情记录、帖子、评论、AI 回复等数据是否正确保存。

```text
loginUser
token
```

## 17. 验收标准

当前阶段验收标准：

1. 用户可以成功注册。
2. 用户可以成功登录。
3. 登录成功后 localStorage 存在 loginUser 和 token。
4. loginUser 中不包含 password。
5. 请求心情记录接口时请求头自动携带 token。
6. 用户可以新增心情记录。
7. 用户可以查询自己的心情记录。
8. 用户可以删除自己的心情记录。
9. 不传 userId 也能查询当前用户记录。
10. 退出登录后 token 被清除。
11. 未登录访问首页或受保护接口会被拦截或提示。
12. 接口返回结构统一为 Result。
13. 系统异常能被全局异常处理捕获。
14. 新增和修改心情记录可以生成 AI 回复。
15. AI 回复可以在列表和详情中展示。
16. AI 调用记录可以保存到 ai_reply 表。
17. AI 调用失败时记录流程不中断。
18. 最近 7 天心情趋势展示正常。
19. 前端主要操作具备 message、loading、empty 状态。

后续完整系统验收标准：

1. 用户可以发布匿名帖子。
2. 用户可以评论、点赞和收藏帖子。
3. 管理员可以管理用户、帖子、评论和话题。
4. 管理员可以查看 AI 回复记录。
5. 用户端更完整的情绪统计图表展示正常。
6. 管理端平台统计展示正常。
7. 分页和条件查询正常。
8. 权限控制有效。
9. 项目可以按部署文档正常运行。

## 18. 测试结论

本系统测试需要覆盖功能正确性、接口规范性、权限安全性、异常稳定性和前后端联调完整性。当前阶段重点验证注册登录、JWT、密码加密、心情记录、AI 回复、AI 调用记录、用户端趋势统计和统一返回等用户端闭环功能；后续阶段再逐步覆盖匿名社区、后台管理和更完整的数据统计。

通过以上测试，可以保证系统核心流程可运行、关键权限不被绕过、数据返回结构统一，并为后续功能扩展提供稳定基础。
